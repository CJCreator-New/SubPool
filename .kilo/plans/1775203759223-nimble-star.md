# SubPool Wishlist Enhancement Plan

## Overview
This plan outlines improvements to the wishlist feature in the SubPool application to better align with product requirements and enhance user experience. The focus is on adding market intelligence, improving filtering/search capabilities, and providing better user feedback.

## Current State Analysis
The wishlist feature allows users to:
- Post requests for subscription platforms with budget and urgency
- View open requests filtered by category (Entertainment/Work/AI/All)
- Sort requests by recent/budget/urgent
- Offer slots to fulfill requests
- Cancel their own requests

## Enhancement Opportunities

### 1. Market Intelligence Integration
**Problem**: Users lack pricing context when making wishlist requests
**Solution**: Add savings calculations and market intelligence to wishlist cards
- Display potential savings percentage based on retail vs. budget price
- Show average market price for requested platforms
- Add "Savings Hub" badge for requests below market average

### 2. Advanced Filtering & Search
**Problem**: Limited filtering options make it hard to find relevant requests
**Solution**: Enhance filtering capabilities
- Add text search for platform names and plan descriptions
- Implement price range filtering (min/max budget)
- Add date-based filtering (requested within X days)
- Include platform-specific filtering beyond categories

### 3. Request Analytics & Insights
**Problem**: Users lack visibility into request trends and fulfillment rates
**Solution**: Add analytics dashboard
- Show fulfillment rate by platform/category
- Display average time to fulfillment
- Trending requests (most requested platforms)
- Success rate metrics for hosts

### 4. Enhanced User Experience
**Problem**: Current UI could be more informative and engaging
**Solution**: Improve wishlist interaction
- Add platform savings calculator in request form
- Show requester's fulfillment history/reliability score
- Add ability to edit/update existing requests
- Improve empty states with actionable suggestions
- Add request expiration indicators

### 5. Technical Improvements
**Problem**: Code could be optimized for better maintainability
**Solution**: Refactor and optimize
- Extract wishlist card component for reusability
- Optimize filtering/sorting with useMemo dependencies
- Add proper TypeScript types for wishlist stats
- Implement optimistic UI updates for better responsiveness

## Implementation Priority

### High Priority (Immediate Impact)
1. Add market intelligence to wishlist cards (savings calculation)
2. Enhance filtering with search and price range
3. Improve empty states with actionable guidance

### Medium Priority (User Engagement)
1. Add request analytics dashboard
2. Implement request editing capability
3. Enhance user profiles with reliability scores

### Low Priority (Long-term Value)
1. Advanced analytics and trend prediction
2. Social features (sharing requests, commenting)
3. Integration with external pricing APIs

## Success Metrics
- Increase in wishlist-to-pool conversion rate
- Reduction in average request fulfillment time
- User satisfaction improvement (via feedback)
- Increase in repeat wishlist usage

## Dependencies
- Supabase schema may need updates for new fields
- Potential backend API enhancements for analytics
- Possible updates to useWishlist hook for new data requirements

## Next Steps
1. Finalize enhancement scope with stakeholders
2. Design UI/UX mockups for new features
3. Implement market intelligence features first
4. Roll out enhancements in phases with user testing
